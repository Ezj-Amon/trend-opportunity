# Amazon 一方数据验证流程（暂不使用 SP-API）

## 先看这里：你只需要做什么

打开 `/validation` 后，从第 1 条“可立即验证”的任务开始：

1. 复制已经在 ProductHypothesis 中由人工确认的具体商品查询词。系统不会从新闻标题或类目自动生成查询词。
2. 在对应站点的 Seller Central 中用这个词分别导出“商机探测器”和“品牌分析 → 热门搜索词”两个 CSV。
3. 回到同一条任务，上传这两个原始 CSV。系统负责匹配指标、评分并更新排序。

Amazon 两份 CSV 对应的平台维度由系统自动解析，不需要人工给需求、购买意图或竞争指标打分。CSV 不包含的单位经济性和执行可行性仍需在工作台根据成本、供应链和合规证据补充 1–5 分。查询词尚未准备好的任务会标为“暂时不用处理”。

## 系统会替你完成什么

- 校验商品假设已完成审核、目标站点明确且查询词来自具体实体商品形态。
- 从两份原始 CSV 中精确匹配查询词，自动读取需求、购买意图、竞争、差异化、时机和证据指标。
- 明确保留仍缺少的单位经济性或执行数据，不要求用户猜数。
- 将每次结果独立保存为 MarketEvidence；新证据不会覆盖旧证据快照。
- 只有证据完整、单位经济和证据评分达到门槛、风险为低或中时，创建 ValidatedRecommendation。

主链路 API 为：

```text
POST /api/product-hypotheses/{hypothesis_id}/amazon-raw-import
POST /api/product-hypotheses/{hypothesis_id}/market-evidence
GET  /api/product-hypotheses/{hypothesis_id}/market-evidence
GET  /api/validated-recommendations
```

旧 `opportunity_id` 标准表和旧机会 API 只用于历史数据迁移。

## Seller Central 下载操作

### 商机探测器

搜索商品词后，在 Niche 视图点击“自定义列”，勾选全部 13 列并保存，然后点击结果表右上方“下载”。首轮需要：细分市场、热门搜索词、点击量最多的商品数量、搜索量（过去 360 天/最近 90 天）、增长（过去 180 天/最近 90 天）、售出商品数量、平均销售商品数量、退货率、平均价格和价格范围。

### 品牌分析 · 热门搜索词

筛选相同搜索词，点击“自定义列”，保留全部 16 列并保存；点击“生成下载项”，完成后从页面右上方“下载管理器”下载文件。首轮需要：搜索词、搜索频率排名、热门点击品牌、点击量最高的分类，以及前三个商品的 ASIN、商品名称、点击占比和转化率份额。

“搜索词表现报告”或“搜索查询绩效”主要用于自有品牌/ASIN，不作为新机会初筛的第一批必需数据。

## 高级方式：项目标准化 CSV

这一方式只用于迁移已有历史评分表，日常验证不需要使用。

- 标识：`opportunity_id`、`target_marketplace`、`provider`、`collected_at`、`search_term`
- 原始指标：搜索量/增长、商品数、价格、评分、评论量、头部点击/购买份额、退货率、搜索频率排名、点击/转化份额、到岸成本、Amazon 费用、广告成本率和预估毛利率
- 评分：搜索需求、购买意图、竞争机会、单位经济、差异化、执行、时机、证据完整度
- 审计：`source`、`note`

`source` 可以用 `|` 分隔多个来源。CSV 必须是 UTF-8；从工作台下载的模板已包含 Excel 兼容的 BOM。

## 原始文件兼容范围

当前解析器适配美国站中文 Seller Central 2026 年 7 月导出格式：商机探测器文件前两行是筛选说明，热门搜索词文件第一行是报告条件。解析器按表头定位，不依赖固定行号，并处理百分比小数、空转化份额、带逗号的商品标题和 UTF-8 BOM。

开发新的导出格式适配时只需要原始表头和 1–2 条代表性数据，用来确认元数据行、编码、空值及百分比格式；不需要提供完整业务数据。
